# Фотоархив компании

Корпоративная информационная система для централизованного хранения, поиска и управления цифровыми фотографиями и отсканированными документами.

## Функциональность

- **Аутентификация**: Вход по логину/паролю с защитой от перебора (блокировка на 5 мин после 5 неудачных попыток)
- **Ролевая модель**: Администратор, Архивариус, Сотрудник — с разграничением прав
- **Поиск файлов**: По интервалу дат и ключевым словам (логика OR), с указанием причины попадания в выдачу
- **Загрузка файлов**: Произвольные файлы хранятся как BLOB в SQLite
- **Редактирование метаданных**: Описание, дата оригинала, ключевые слова
- **Управление пользователями**: Смена ролей, блокировка/разблокировка (для Администратора)
- **Журнал событий**: Аудит всех действий пользователей
- **Пагинация**: 10 записей на страницу

## Структура проекта

```
├── app.py                  # Flask-приложение, маршруты, API-эндпоинты
├── database.py             # SQLAlchemy-модели (Role, User, File, Keyword, FileKeyword, EventLog)
├── init_db.py              # Скрипт первичного наполнения (роли + admin/admin)
├── requirements.txt        # Зависимости
├── static/
│   ├── css/style.css       # Стили интерфейса
│   ├── js/main.js          # Fetch API логика (login, role change, delete, block)
│   └── icons/              # SVG-иконки типов файлов
│       ├── pdf.svg
│       ├── docx.svg
│       ├── xlsx.svg
│       ├── image.svg
│       └── default.svg
└── templates/
    ├── base.html            # Базовый шаблон (topbar, профиль, навигация)
    ├── login.html           # Страница входа
    ├── search.html          # Поиск файлов + результаты
    ├── file_detail.html     # Детальная карточка файла
    ├── file_edit.html       # Редактирование метаданных
    ├── upload.html          # Загрузка нового файла
    ├── users.html           # Управление пользователями
    └── logs.html            # Журнал событий
```

## Запуск

```bash
pip install -r requirements.txt
python init_db.py        # Создание таблиц и дефолтного администратора
python app.py            # Запуск сервера
```

После запуска откройте: `http://127.0.0.1:5000`

Учётные данные по умолчанию: **admin / admin**

## API-эндпоинты

| Метод | Эндпоинт | Описание | Роли |
|-------|----------|----------|------|
| POST | `/api/auth/login` | Вход в систему | Все |
| POST | `/api/auth/logout` | Выход из системы | Все |
| GET | `/api/files/download/<id>` | Скачивание файла | Авторизованные |
| DELETE | `/api/files/delete/<id>` | Удаление файла | Администратор |
| PUT | `/api/admin/users/<id>/role` | Смена роли | Администратор |
| PUT | `/api/admin/users/<id>/status` | Блокировка/разблокировка | Администратор |
| GET | `/api/admin/logs` | Журнал событий (JSON) | Администратор |
